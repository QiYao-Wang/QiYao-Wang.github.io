OpenSeeker-v2：以高质量高难度轨迹推动搜索Agent极限 —— 仅用SFT超越工业级CPT+SFT+RL训练管线

📅 June 07, 2026 · AI-assistanted Qiyao Wang #Search Agent #Paper

本文介绍上海交通大学团队的最新工作 OpenSeeker-v2——一个全新的搜索 Agent，仅用 10.6K 高质量 SFT 数据即取得 BrowseComp 46.0%、BrowseComp-ZH 58.1%、HLE 34.6%、xbench 78.0% 的 SOTA 成绩，全面超越通义 DeepResearch（CPT+SFT+RL）和 RedSearcher。它是首个在该模型规模和范式（ReAct）下由纯学术团队仅用 SFT实现的最先进搜索 Agent。

  • 核心贡献：揭示了当 SFT 数据具备足够的高难度和高信息密度时，简单 SFT 即能训练出超越复杂多阶段工业管线的搜索 Agent
  • 三项简单数据合成改进：(1) 扩大知识图谱规模以增加探索丰富度，(2) 扩展工具集以增强功能广度，(3) 严格低步数过滤以确保最低难度底线
  • 仅 10.6K 样本通过单次 SFT 训练，在四个代表性基准全面超越同规模 ReAct Agent，并在 BrowseComp-ZH 上 领先通义 DeepResearch 11.4 个百分点
  • 数据难度碾压：OpenSeeker-v2 平均每轨迹 64.67 步，远超 v1 的 46.97 步和 RedSearcher 的 36.01 步
  • 首个纯学术 SOTA：证明学术团队可通过战略性数据合成绕过工业界的「数据护城河」，完全开源模型权重

Motivation：搜索 Agent 的「重训练管线」迷思

深度搜索能力已成为前沿大语言模型 Agent 不可或缺的核心竞争力，但其发展长期被资金雄厚的企业巨头垄断。典型的工业界 SOTA 方案依赖于极其资源密集的训练管线：持续预训练（CPT）→ 监督微调（SFT）→ 强化学习（RL）。通义 DeepResearch、RedSearcher 等代表性工作均采用这一重型管线，需要海量计算资源和专有数据管道，对学术界和开源社区形成了巨大的参与壁垒。

核心问题：我们是否真的需要 CPT+SFT+RL 这样的重型训练管线？能否仅用简单的 SFT 方法，通过提升训练轨迹的难度和信息丰富度，推动搜索 Agent 达到与重型工业管线相媲美的性能？

在 OpenSeeker-v1 的基础上，OpenSeeker-v2 将焦点完全转向训练轨迹本身的质量，提出了一个令人惊讶的发现：当 SFT 数据具备足够的高难度和丰富信息时，简单 SFT 就可以足够强大，甚至超越重型工业管线。

方法：三项简单但高效的数据合成改进

OpenSeeker-v2 的核心假设是：给定足够困难和信息丰富的训练数据，直接的 SFT 目标足以引导模型学习强大的长程搜索和推理能力。为此，作者在 v1 的数据合成 pipeline 上引入了三项看似简单但极其有效的修改：

改进一：扩大知识图谱规模，注入更丰富的探索空间

设 \(\mathcal{G}=(\mathcal{V},\mathcal{E})\) 表示用于任务合成的源知识图谱。在原始 pipeline 中，从种子节点 \(v_{\mathrm{seed}}\) 出发沿出边遍历 \(k\) 个相邻节点，构建局部子图。而在 v2 中，将扩展预算从 \(k\) 提升至 \(K\)（\(K > k\)）：

\[ \mathcal{G}_{\mathrm{sub}}^{(K)} = \operatorname{Expand}(\mathcal{G}, v_{\mathrm{seed}}, K) \]

扩大后的子图包含更丰富的拓扑相关源信息，增加了可行推理路径的数量和多样性。合成问题随后基于这个扩展上下文生成：

\[ q \sim P_{\mathrm{gen}}\left(q \mid \mathcal{G}_{\mathrm{sub}}^{(K)}\right) \]

通过增大 \(K\)，生成的问题更可能需要跨多个节点的证据聚合，而非仅依赖少数源即可解决。这一步的核心作用是结构性地强制多跳探索，使得训练出的 Agent 天然具备深度搜索的能力。

改进二：扩展工具集，赋予更广泛的功能覆盖

给定生成的问题 \(q\)，v2 为搜索 Agent 配备了比 v1 更大的工具集 \(\mathcal{A}\)，让其产生多步 ReAct 风格轨迹：

\[ \tau = \left( r_1,a_1,o_1, r_2,a_2,o_2, \ldots, r_T,a_T,o_T, r_{T+1},y \right) \]

其中每个动作 \(a_t \in \mathcal{A}\) 对应一次工具调用，\(o_t\) 表示工具返回的观测，\(r_t\) 为每步动作前的推理痕迹。轨迹包含 \(T\) 步工具调用，随后进行最终推理 \(r_{T+1}\) 并给出答案 \(y\)。

通过扩展 \(\mathcal{A}\)，Agent 被鼓励学习更多样化的交互模式，并利用互补工具组合，产生更灵活、功能更丰富的问题解决行为。这直接提升了 Agent 在真实场景中应对多样化查询的能力。

改进三：严格低步数过滤，确保最低难度底线

为移除过于简单的训练实例，OpenSeeker-v2 引入严格的低步数过滤规则：

\[ \mathcal{D}_{\mathrm{v2}} = \left\{ (q,\tau) \in \mathcal{D}_{\mathrm{raw}} \;\middle|\; T(\tau) \geq T_{\min} \right\} \]

其中 \(T_{\min}\) 为预设的最低工具调用阈值。工具调用步数 \(T(\tau) < T_{\min}\) 的轨迹被直接丢弃，因为它们往往可以通过直接查表或浅层关键词匹配解决，无法为模型提供有效的长程推理训练信号。

三项改进的协同效应：扩大图谱增加了上下文丰富度和多跳依赖性 → 扩展工具集赋能更多样化的解决策略 → 低步数过滤强制执行最低难度底线。三者共同作用，产出了高质量 SFT 数据，驱动 Agent 学会持续推理、鲁棒信息提取和长程搜索行为。

实验设置

训练与评估配置

模型初始化：基于 Qwen3-30B-A3B-Thinking-2507 初始化（总参 30B，推理时激活 3B），256K 上下文窗口，最多 200 次工具调用。

训练方式：仅 SFT，无 RL，无额外超参调优。仅用 10.6K 条高难度轨迹进行单次训练。

评估基准：四个代表性 Agent 基准——BrowseComp（英文多步搜索）、BrowseComp-ZH（中文版）、Humanity's Last Exam（HLE）（跨领域高难度问答）、xbench-DeepSearch（深度研究能力评估）。

对比基线：涵盖三大类——闭源专有模型（Claude-4-Opus、GPT-5-High 等）、开源大模型（DeepSeek-V3.2-671B、GLM-4.7-357B 等）、以及同规模 ~30B 的 ReAct 搜索 Agent（Tongyi DeepResearch、RedSearcher、WebSailor-V2 等），后者是论文的核心对比对象。

结果分析

超越重型工业管线的 SFT

OpenSeeker-v2 的核心问题是：简单 SFT 能否推动搜索 Agent 达到极限并媲美重型工业管线？答案是肯定的。

① 全面超越同规模 CPT+SFT+RL Agent：在同为 ~30B 的 ReAct 搜索 Agent 中，OpenSeeker-v2-30B-SFT 以仅 SFT 训练取得了最强的综合性能。在 BrowseComp 和 HLE 上分别比通义 DeepResearch 和 RedSearcher 高出至少 2.6% 和 0.3%；在 BrowseComp-ZH 和 xbench 上更是分别领先通义 DeepResearch 11.4% 和 3%。

② 超越更大规模的开源模型：OpenSeeker-v2 甚至超越了参数规模大得多的模型，包括 DeepSeek-V3.1-671B、GLM-4.6-357B、Minimax-M2-230B 以及 Claude-4.5-Sonnet，充分说明数据质量可以弥补参数规模差距。

③ v1 → v2 的显著提升：在相同模型规模和 SFT-only 训练设定下，v2 相比 v1 实现了巨大跃迁——BrowseComp 从 29.5→46.0（+56%），BrowseComp-ZH 从 48.4→58.1（+20%），xbench 从 74.0→78.0。这表明 OpenSeeker 框架在 SFT 设定下远未饱和，提升数据质量即可持续获得性能增益。

数据难度：v2 的长轨迹优势

OpenSeeker-v2 的训练数据在难度上远超同类工作，这是其性能优势的根本来源。

OpenSeeker-v2 的平均每轨迹工具调用次数达到 64.67 步，远超 OpenSeeker-v1 的 46.97 步和 RedSearcher 的 36.01 步。这一显著差距说明 v2 的训练数据要求更复杂的多步推理和更长程的信息搜寻。作者假设，这种长而困难的人工合成轨迹对于使模型获得更强的长程检索和搜索能力至关重要，这也解释了为何 v2 在挑战性 deep research 基准上表现优异。

关键洞察：扩大图谱 → 增加多跳依赖 → 轨迹变长 → 难度提升 → Agent 学会真正的长程搜索，而非走捷径。这一链条揭示了「数据难度驱动能力提升」的内在机制。

OpenSeeker 框架的可扩展性

v1 → v2 的性能跃迁表明，通过提升合成 QA 任务的难度和丰富度、增强合成轨迹的整体质量，可以带来实质性的能力增益。这为 OpenSeeker 框架的持续发展指明了方向：可扩展的高质量数据合成是推动搜索 Agent 前进的有希望路径。

作者的内部观察表明，高质量合成数据展现出强大的扩展潜力。未来工作将继续沿此方向推进——扩大数据量、提升数据质量和多样性，以进一步推动搜索 Agent 的性能边界。

总结

OpenSeeker-v2 的核心贡献可以归纳为：

① 用简单 SFT 打破「重型训练管线」迷思：证明了当数据质量足够高时，简单 SFT 即可训练出超越 CPT+SFT+RL 重型管线的搜索 Agent。这一发现从根本上挑战了工业界对复杂多阶段训练的路径依赖。

② 三项简单但高效的数据改进：扩大知识图谱规模以增加探索丰富度、扩展工具集以增强功能广度、严格低步数过滤以确保难度底线——三项修改的组合产生了显著的性能提升，且实现简单、易于复现。

③ 首个纯学术 SOTA 搜索 Agent：OpenSeeker-v2 是首个在模型规模和范式（ReAct）内由纯学术团队仅用 SFT 实现的最先进搜索 Agent。团队完全开源模型权重，使前沿搜索 Agent 研究对学术界和开源社区更加可及。

④ 数据质量 > 训练复杂度：仅用 10.6K 高质量样本 + 简单 SFT，全面超越了使用 CPT+SFT+RL 训练的工业界方案。这强有力地证明了：精心设计的数据本身就足以解锁巨大的性能增益。

核心启示：在搜索 Agent 领域，数据质量可能是训练智能长程搜索 Agent 的关键路径。与其堆叠复杂的多阶段训练管线（CPT → SFT → RL），不如将精力集中在提升训练数据的难度和信息丰富度上。OpenSeeker-v2 用事实证明：数据质量优先于训练规模。
